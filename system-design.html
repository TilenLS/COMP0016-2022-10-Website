System Design

System Architecture

Pyside and Open3d is used for the frontend, to build the user interface and to view tooth models in PLY file format. For the backend, SQLite is used to store user's credentials and patient's information. Deep learning models on classification for tooth wear grading and segmentation for meta tagging is implemented with Pytorch. Besides, we are using PointNet, which is a robust algorithm on 3D cloud point classification and segmentation for the neural network.

This is the latest system architecture diagram which is more concise and appropriate. We have moved our database to Microsoft Azure, where Python Requests library and Flask is used to send and receive data.

Database Structure

The database consists of two tables. The first table is called "Dentists", it stores user's credentials such as id, username and password. Authorised users to login and view patient tooth models are dentists, hence the table is named "Dentists". ID is automatically created and incremented every time a new user registers.

The second table is called "Patients", it stores patient's information that are relevant to decide tooth wear degree and generate treatment plans. Patient's tooth model is stored using file path.
